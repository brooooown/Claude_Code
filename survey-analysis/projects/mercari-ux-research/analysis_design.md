# Mercari UX Research - 分析設計書

## 分析アプローチ概要

### 1. **フォロー行動深掘り分析**

#### 1.1 量的分析
- **クロス集計分析**: フォローセグメント × バイヤーセグメント
- **相関分析**: フォロー頻度と購買行動の関係
- **因子分析**: フォロー動機の要因構造
- **カイ二乗検定**: セグメント間の独立性検証

#### 1.2 質的分析
- **テキストマイニング**: フォロー理由の自由回答分析
- **カテゴリー分類**: フォロー動機のグルーピング
- **感情分析**: フォローに対するポジティブ/ネガティブ要因

### 2. **セグメント別戦略提案分析**

#### 2.1 セグメントプロファイリング
- **記述統計**: 各セグメントの基本特徴
- **行動パターン分析**: セグメント別機能利用状況
- **価値観分析**: セグメント別重要度指標

#### 2.2 戦略示唆導出
- **RFM分析風アプローチ**: Recency, Frequency, Monetary価値のMercari版
- **ニーズギャップ分析**: 現在利用vs重要度のマッピング
- **ペルソナ構築**: 代表的ユーザー像の構築

## 分析工程設計

### Phase 1: データ理解・前処理
**期間**: 1日目
**成果物**: データ品質レポート、クリーニング済みデータ

1. データ概要把握
2. 欠損値・異常値処理
3. カテゴリ変数の整理
4. 新指標の算出

### Phase 2: 基本分析・セグメント分析
**期間**: 1-2日目
**成果物**: 基本統計レポート、セグメント特徴分析

1. 記述統計（デモグラフィック）
2. セグメント分布とクロス分析
3. セグメント別特徴抽出
4. 初期インサイト導出

### Phase 3: フォロー行動深掘り分析
**期間**: 2-3日目
**成果物**: フォロー行動詳細レポート

1. フォロー動機のテキスト分析
2. フォロー頻度と購買の関係性分析
3. フォローセグメント別深掘り
4. 統計的検定による仮説検証

### Phase 4: 統合分析・戦略提案
**期間**: 3-4日目
**成果物**: 最終戦略提案書

1. 各分析結果の統合
2. セグメント別ペルソナ構築
3. 戦略的示唆の導出
4. アクションプラン策定

## 使用する分析手法詳細

### 統計手法
```
- 記述統計: mean, median, mode, std
- クロス集計: pd.crosstab, chi2_contingency
- 相関分析: pearson, spearman
- 因子分析: PCA, Factor Analysis
- クラスタリング: K-means, Hierarchical
```

### 可視化手法
```
- 基本グラフ: bar, pie, histogram, box plot
- 高度な可視化: heatmap, scatter matrix, radar chart
- セグメント比較: parallel coordinates, bubble chart
- ネットワーク図: フォロー関係の可視化
```

### テキスト分析
```
- 形態素解析: MeCab/janome
- 頻度分析: TF-IDF
- 感情分析: sentiment analysis
- トピック抽出: LDA
```

## データ処理方針

### 前処理ルール
1. **欠損値処理**:
   - 数値: 平均値/中央値補間
   - カテゴリ: 「不明」カテゴリ作成
   - 重要項目: 欠損率5%以上は要注意

2. **外れ値処理**:
   - 四分位範囲(IQR)による検出
   - 業務ロジックによる妥当性確認
   - 除外 vs 変換の判断基準設定

3. **カテゴリ変数**:
   - レスポンス系項目の統合
   - 頻度の少ないカテゴリの統合
   - ダミー変数化の方針

## 品質保証

### 各Phase終了時の確認点
1. **データ品質**: 処理前後の整合性確認
2. **分析妥当性**: 統計的前提条件の確認
3. **解釈妥当性**: ビジネス観点での妥当性確認
4. **再現性**: コード・データの保存確認
